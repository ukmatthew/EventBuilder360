## Detailed Run of Show

### 0-10 min - Opening + Credibility
**Slides 1-3**
- Set expectation: "This is not a talk. You're going to build, think, and explain."
- Show real product proof (built solo, complex system, modular thinking).
- Share "newbie to launch" evolution and first-principles mindset.

### 10-20 min - Definition + Process
**Slides 4-5**
- Define what Vibe Coding is vs is not.
- Anchor the loop: **Explain -> Plan -> Build -> Observe -> Refine**.
- Core line: "AI reflects the clarity you give it."

### 20-22 min - Intermission Reset
**Slide 6**
- Quick reset and transition into practical workshop flow.

### 22-35 min - Vision to Practical Framing
**Slides 7-10**
- Present target state and why overwhelm is normal.
- Reframe to plan-first, modular execution.
- Enter "Plan Well" section.
- Live walkthrough of `.cursorrules`, `/planning`, and architecture context.

### 35-45 min - LEGO Warm-Up
**Slide 11**
- Build a tower (**3 min**, include minifigure).
- Each person explains one special thing (**~30 sec each**).
- Purpose: warm hands + practice concise explanation.

### 45-65 min - Problem Ideation Activity
**Slides 12-13**
- Explore "what drives you mad?" across event-app pain points.
- Structured flow:
  - Build LEGO model of the issue (**5 min**)
  - Explain model at table (**~1 min each**)
  - Table picks favorite problem (**2 min**)
  - Room share-out of chosen table problems

### 65-90 min - Build Modular (Live)
**Slides 14-19**
- Show starter app baseline and current task.
- Teach and use the **6-step prompt structure**.
- Live handoff to AI ("Over to you Claude/ChatGPT").
- Demonstrate bad-context failure and recovery.
- Collaborative mode: build prompt together with audience input.

### 90-105 min - Keep Planning
**Slides 20-21**
- Roadmap prioritization phase and live capture.
- Convert activity insights into actionable Vibe Coding tasks.

### 105-120 min - Close
**Slides 22-25**
- Closing remarks.
- Reinforce principle: "Before code, there is explanation."
- Final takeaway: "Start with a plan. Anchor the AI. Build one block at a time."
- Mission complete: repo/resources/community CTA.

## Facilitator Timing Notes
- Keep momentum tight through slides 1-10; avoid deep rabbit holes early.
- Protect activity windows (Slides 11-13) so participants feel the workshop is hands-on.
- In Slides 14-19, prioritize one concrete "bounded win" over broad feature scope.
- If behind time, compress discussion on Slides 20-21 and move to close sequence.
